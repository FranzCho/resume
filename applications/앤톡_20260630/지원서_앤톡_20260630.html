SELF-INTRODUCTION · 자기소개서
01
지원동기

지난 2년 5개월간 ㈜마이아이비에서 기부·모금·봉사 플랫폼의 프론트엔드를 담당하며, 신뢰가 흔들리는 순간 사용자가 바로 떠난다는 사실을 매일 코드로 마주했습니다. 후원금이 한 번 잘못 결제되거나 단체가 부정확한 정보로 노출되면 플랫폼 전체의 신뢰가 흔들립니다. 그래서 화면 한 장의 가설을 PRD로 정의하는 일부터, 디자인 시스템 토큰화와 Sentry 누적 에러 전수 조사로 그 화면이 안정적으로 돌아가게 만드는 일까지를 한 사람이 이어서 책임지는 방식으로 일해 왔습니다.

앤톡을 지원한 이유는 "빅데이터로 국내 기업 생태계를 전산화하고 더 정교하고 차별적인 분석정보를 제공한다"는 회사의 방향이 제가 myib에서 익힌 일하는 방식과 맞닿기 때문입니다. 기업 정보 관리 내부 서비스와 분석 대시보드·리포트처럼 기업 데이터와 분석 결과가 화면 한 장의 정합성으로 사용자에게 전달되는 영역에서는, 제품 표면의 안정성과 데이터 정합성이 그대로 사용자 신뢰의 근거가 된다고 생각합니다. 임팩트 도메인에서 쌓은 신뢰 중심의 프론트엔드 설계·운영 경험을 바탕으로, 핀테크 데이터 영역에서 앤톡의 제품 가치를 함께 고도화하고자 지원했습니다.

02
직무 경험 ① — 디자인 시스템을 코드에서 끌어올린 1년

myib 입사 시 코드베이스는 hex 컬러가 수백 곳에 흩어져 있고, native alert() 호출이 58개, 어드민 테이블이 6종 비표준으로 운영되고 있었습니다. 새 컴포넌트를 만들기 전에 코드에 이미 형성된 패턴을 카테고리별로 추출하는 bottom-up 접근을 택했고, 약 1년에 걸쳐 banner·button·icon·selector·table·toast 등 20여 카테고리를 토큰화한 뒤 Storybook을 GitHub Pages에 자동 배포되도록 CI를 세팅했습니다. 디자이너·개발자가 매번 합의하지 않아도 되도록 Figma↔코드 매핑을 단일 진실 문서(DESIGN.md)로 묶었습니다.

특히 데이터 중심 화면을 표준화한 경험이 앤톡 업무와 직접 맞닿습니다. 공통 컴포넌트 DataTable·StatusBadge를 신규 설계해 내부 어드민(admin2)의 26개 테이블 중 22개를 표준 교체했고, 흩어진 native alert 58건을 일관된 showToast 패턴으로 통일했습니다. 또 레거시 admin 190파일을 단일 commit으로 이관하고 미사용 컴포넌트·정적 파일 320여 개를 회귀 없이 정리, 아이콘 100여 개를 60개로 통합(40%↓)하며 버튼을 FillButton 단일 표준으로 90개 파일 일괄 교체해 코드베이스 부피와 빌드 비용을 함께 줄였습니다. 앤톡이 우대하는 TypeScript·Tailwind CSS·Styled-components 기반 디자인 시스템과 공통 UI 컴포넌트 구축은 이 경험을 그대로 옮길 수 있는 영역이라고 봅니다.

03
직무 경험 ② — 기획서를 직접 쓰고 그 위에서 코드를 짠다

프론트엔드 개발자로 일해왔지만, 화면 한 장을 만들 때마다 사용자가 어디서 들어와 무엇을 결정하고 어디로 빠져나가는지를 먼저 그려보는 사람입니다. 기능 명세를 받아 구현하기보다, 유저의 의사결정 동선·고객사의 노출 요구·웹에서의 표현 방식 세 축을 한 문서에서 함께 설계한 뒤 직접 구현하는 사이클로 일합니다. 대표 사례인 단체 상세 페이지 리디자인 PRD에서는 CS 응대 기록을 출발점으로 '신뢰 신호 분산·CTA 휘발성·임팩트 추상' 세 문제를 정의하고, 진입·검토·결정·결제로 이어지는 퍼널 전체에서 North Star와 7가지 하위 KPI를 잡은 뒤 7개 화면 인터랙티브 프로토타입으로 옮겨 동료들이 클릭만으로 가설을 검토하게 했습니다.

데이터를 사용자가 직관적으로 이해하도록 화면을 다시 짠 경험도 있습니다. 마이페이지 정기기부 화면을 단순 탭형에서 통계 카드·구독 리스트·상태 배너를 갖춘 대시보드형으로 재설계하고, 기부·포인트·매칭 내역 3종을 실데이터로 통합했습니다(9개 메뉴 × 5개 상태, 약 45조합 정리). 기능 구현을 넘어 정보 구조와 시각적 완성도까지 함께 끌어올리는 작업 방식은, 기업 데이터·분석 결과·리포트를 쉽고 직관적으로 이해할 수 있는 화면으로 만든다는 앤톡의 방향과 정확히 맞닿습니다. 화면 위 가설부터 하부 데이터·관측성 레이어의 정합성(Apollo Client v3→v4 마이그레이션, deprecated 쿼리 10건 이관, Sentry 누적 에러 23건 조사)까지 이어서 책임지는 점도, 데이터 중심 서비스에 꼭 필요한 역량이라고 봅니다.

04
성격과 일하는 방식 — 정밀함·완결성과 협업, AI 레버리지

성격의 가장 큰 특징은 모호한 현상에 머무르지 않고 근본 원인을 추적해 체계화하는 정밀함과 완결성입니다. 컴포넌트 배치 오차를 사용처마다 스타일 래퍼로 봉합하는 대신 SVG의 viewBox를 통일해 한 번에 해결하는 것처럼, 한 번 정립한 구조적 원칙을 코드베이스 전체에 일관되게 적용합니다. 예외 처리 미비로 자라던 Sentry 누적 에러 23건을 하루 만에 전수 조사·보강한 것도 같은 성향에서 비롯되었습니다.

이 정밀함은 협업에서 말로 앞서기보다 작동하는 시스템과 정제된 문장으로 소통하는 방식으로 발현됩니다. 1인 프론트엔드로 디자이너·백엔드·기획·데이터 직군과 협업하며, 서로의 언어가 달라 생기는 싱크 오류를 줄이는 데 집중했습니다. 한편 시스템 전체의 큰 그림을 먼저 조망하려는 성향이 있어 초기 설계가 신중해질 수 있는데, 이를 보완하려고 두 가지 레버리지를 씁니다. 첫째는 고속 실행이 필요한 타이밍에 업무 모드를 전환하는 메타인지이고, 둘째는 Claude·Gemini 같은 AI 도구를 가상 아키텍처 파트너로 워크플로우에 내재화하는 능력입니다. 다만 기업·금융 데이터의 정확성과 사용자 신뢰가 최우선인 도메인 특성을 고려해, AI가 제안한 구조나 데이터 치환 결과를 무비판적으로 수용하지 않고 철저한 교차 검증으로 할루시네이션·결함 리스크를 통제합니다.

05
입사 후 포부

앤톡에서 함께 일할 기회를 주신다면, 제 주장을 앞세우기보다 팀의 소통 방식과 기존 코드베이스의 문화를 먼저 온전히 흡수하는 데 집중하겠습니다. 기업 정보 관리 내부 서비스와 분석 대시보드·리포트 등 기업 데이터를 사용자에게 전달하는 핵심 서비스에 성실히 적응하여, 기존 팀원들이 쌓아 올린 토대 위에 묵묵히 제 몫의 개발 리소스를 더하겠습니다. 핀테크 데이터 도메인의 특성상 기업 데이터·분석 결과·시각화가 한 화면 위에서 동시에 흐르는 만큼, 자만하지 않고 기초부터 철저히 배우겠습니다.

이후에는 제가 가진 경험이 팀의 업무 부담을 실질적으로 더는 무기가 되기를 바랍니다. 제로베이스부터 시스템을 단독 구축하고 PRD를 주도적으로 다듬어 온 경험, 그리고 데이터 중심 화면을 대시보드형으로 재설계해 온 경험은 빅데이터 기반 기업 분석 정보를 내외부 서비스와 데이터 시각화로 확장하는 단계에서 그대로 옮겨갈 자산이라 봅니다. 평소 연구해 온 AI 협업 워크플로우 역시 혼자만의 생산성에 머무르지 않고, 팀 전체의 개발 손길을 가볍게 만드는 데 조용히 기여하고 싶습니다.

"금융시장의 새로운 물결을 일으킨다"는 앤톡의 방향, 그리고 빅데이터로 기업 생태계를 전산화한다는 미션은 결코 혼자서 만들 수 없다고 믿습니다. 팀원들과 긴밀히 협업하고 끊임없이 배우며, 앤톡이 지향하는 가치를 사용자가 마주하는 화면에서 완벽한 신뢰성으로 구현해 나가는 가장 성실하고 든든한 동료가 되겠습니다.
