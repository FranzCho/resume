## 1. 나의 업무 스타일 및 장점과 단점

저의 가장 큰 장점은 **모호한 현상에 머무르지 않고 근본적인 원인을 추적해 체계화하는 정밀함과 완결성**입니다. 컴포넌트 배치 오차를 사용처에서 스타일 래퍼로 임시 봉합하는 대신 SVG 자체의 viewBox를 24×24로 통일해 해결하는 것처럼, 한 번 정립한 구조적 원칙을 코드베이스 전체에 일관되게 적용하려 합니다. myib에서 예외 처리 미비로 자라고 있던 Sentry 누적 에러 23건을 하루 만에 전수 조사하고 정리한 것도 같은 성향에서 비롯되었습니다.

이러한 정밀한 성격은 협업에서 **말로 앞서기보다 작동하는 시스템과 정제된 문장으로 소통하는 방식**으로 발현됩니다. 1인 프론트엔드 개발자로서 디자이너·백엔드·기획·마케팅 등 다양한 직군과 협업하며, 서로의 언어가 달라 발생하는 싱크 오류를 줄이는 데 집중해 왔습니다. 디자이너와의 커뮤니케이션 비용을 줄이기 위해 Figma와 코드를 일원화한 단일 진실 문서(DESIGN.md)를 도입하고, 백엔드 개발자와의 데이터 규격을 맞추기 위해 PRD에 엔지니어링 주석을 달아 사전에 합의하는 방식을 자리잡게 했습니다.

또 하나의 핵심 작업 방식은 **AI 도구(Claude, Gemini)를 가상 아키텍처 파트너로 워크플로우에 내재화**하는 능력입니다. 기획서 초안 단계에서 사용자 동선과 페르소나를 빠르게 정리하고, 디자인 시스템 작업에서는 수백 곳에 흩어진 hex 치환을 일괄 처리하며, 작업 컨텍스트와 의사결정 근거를 메모리로 누적해 같은 판단을 반복하지 않도록 관리합니다. 단, AI가 제안한 구조나 데이터 치환 결과물을 무비판적으로 수용하지 않고, 정밀한 성향을 바탕으로 교차 검증 단계를 거쳐 기술적 할루시네이션이나 결함 리스크를 통제합니다.

**단점은 시스템 전체의 구조와 큰 그림을 먼저 조망하려는 성향**에서 옵니다. 새로운 코드베이스에 들어갈 때 머릿속에 큰 그림이 그려지기 전까지는 첫 손이 늦은 편이어서, 진입 단계의 속도가 더디게 느껴질 수 있습니다. 또 한 사람이 여러 표면을 동시에 보는 환경에 익숙해진 만큼, 한 영역의 전문성을 더 깊이 파고드는 시간을 의도적으로 만들어야 한다는 점도 보완해야 할 부분으로 보고 있습니다. 이를 위해 *'정돈은 점진적으로, 0→1 신규 셋업은 의도적으로 빠르게'* 두 가지 모드를 명확히 구분해 일하고 있으며, myib에서 B2B 파트너 프론트엔드 셋업이 단기간에 캠페인·프로필·계좌 도메인까지 출시된 것은 이 시점에는 진입의 신중함보다 추진력이 필요하다는 판단이었습니다.

## 2. 내가 이 포지션에 적합한 사람인 이유

아미쿠스렉스가 찾는 *"Next.js 기반 B2B 웹 애플리케이션 개발 + 디자인 시스템 컴포넌트 제작(Radix UI + Tailwind CSS) + AI 기반 기능 구현"* 세 축은 제가 myib에서 누적해 온 작업 영역과 거의 그대로 일치합니다.

**① Next.js + TypeScript 기반 B2B 시스템 0→1 개발 경험**

myib에서 B2B 파트너 프론트엔드 시스템을 Next.js 16 기반으로 제로 베이스에서 단독 셋업했습니다. Tailwind CSS 컬러 토큰과 빌드 구성을 메인 웹앱 디자인 시스템과 결합하여 멀티 레포지토리 간 자산 호환성을 확보했고, 단체 회원 가입·인증, 캠페인 등록·관리, 프로필 관리, 계좌·정산 도메인을 0→1로 출시했습니다. Next.js의 Pages Router와 App Router 양쪽을 모두 다뤘으며, generateMetadata API를 활용해 도메인별 layout 10개 영역에 SSR 기반 메타데이터를 적용한 경험도 있습니다.

**② 디자인 시스템 0→1 구축 및 컴포넌트 라이브러리 표준화**

myib 입사 시 코드베이스는 hex 컬러가 수백 곳에 흩어져 있고, native alert() 호출이 58개, 어드민 테이블 6종이 비표준으로 운영되고 있었습니다. 약 1년에 걸쳐 banner·button·icon·selector·table·toast 등 20여 개 카테고리를 토큰화하고, Storybook을 구축해 GitHub Pages에 자동 배포되도록 CI를 세팅했습니다. 공통 컴포넌트 DataTable·StatusBadge를 신규 설계해 내부 어드민(admin2)의 26개 테이블 중 22개를 표준 교체했고, FillButton 단일 표준화로 90개 파일의 결합 구조를 일괄 교체했습니다. 또한 레거시 admin 디렉토리 190개 파일을 외부 의존성과 함께 단일 commit으로 이관하고, 미사용 컴포넌트·유틸·정적 파일 320여 개를 일괄 정리하면서 코드베이스의 부피와 빌드 비용을 함께 줄였습니다. 아미쿠스렉스의 *"Radix UI + Tailwind CSS 기반 디자인 시스템 컴포넌트 제작"* 업무는 이 경험을 그대로 옮길 수 있는 영역이라고 봅니다.

**③ AI 도구 활용 + AI 기반 인터페이스 구현 경험**

myib에서 모금 만들기 화면의 LLM 기반 AI 자동채우기 기능을 개선·연동하면서 AI가 출력한 콘텐츠를 사용자가 검토·수정·재요청할 수 있는 인터페이스 패턴을 다뤘습니다. 더 넓게는 평소 Claude·Gemini를 가상 아키텍처 파트너로 워크플로우에 내재화하여 대량의 레거시 타입 치환을 가속화하고, 교차 검증 단계를 거쳐 할루시네이션·결함 리스크를 통제하는 작업 방식을 정착시켰습니다. 아미쿠스렉스의 *"AI 도구를 활용한 개발 생산성 향상 + AI 기반 기능 구현"* 미션과 직접 연결됩니다.

**④ 상태관리·협업·API 경계**

상태 관리는 myib에서 Zustand + Jotai + React Context를 도메인별로 분리해 사용해 왔으며, Apollo Client v3 → v4 마이그레이션을 단독 수행한 경험이 있습니다. 백엔드 담당자와 협업해 deprecated GraphQL 쿼리·필드 10건을 식별하고 신규 스키마로 안전하게 이관하면서, 프론트엔드와 백엔드 경계를 유연하게 넘나드는 작업도 익숙해졌습니다. Git 협업은 4개 레포지토리에서 약 2,200+ 커밋, PR/코드 리뷰/브랜치 전략을 통해 자리잡았습니다.

**⑤ CKEditor 등 에디터 운영 경험 (우대사항 매칭)**

myib에서 CKEditor 5 기반 본문 에디터를 운영 및 커스터마이징한 경험이 있습니다. 리치 텍스트 에디터의 콘텐츠 모델·플러그인 구조·이미지 업로드 파이프라인을 다뤄본 만큼, 로폼의 *"법률문서 자동작성"* 영역에서 요구되는 에디터 커스터마이징 작업에도 빠르게 진입할 수 있다고 봅니다.

## 3. 나의 지원 동기 및 포부

지난 2년간 myib에서 기부·모금·봉사 플랫폼의 프론트엔드를 담당하며, **신뢰가 흔들리는 순간 사용자가 바로 떠난다**는 사실을 매일 코드로 마주했습니다. 후원금이 한 번 잘못 결제되면 사용자는 다시 돌아오지 않고, 단체가 한 번 부정확한 정보로 노출되면 플랫폼 전체에 대한 신뢰가 영향을 받습니다. 그래서 단체 상세 페이지 PRD 작성부터 디자인 시스템 토큰화, Sentry 누적 에러 전수 조사까지 화면 한 장의 가설을 정의하는 일부터 그 화면이 안정적으로 돌아가게 만드는 일까지를 한 사람이 이어서 책임지는 방식으로 일해 왔습니다.

아미쿠스렉스를 지원한 이유는 **"누구에게나 법의 가치가 실현되는 세상을 만든다"**는 로폼의 비전이 제가 myib에서 익힌 일하는 방식과 결을 같이 한다고 느꼈기 때문입니다. 법률문서는 한 줄의 오타가 사용자의 권리·자산·계약 관계 전체에 영향을 미칠 수 있는 영역입니다. 신뢰가 무너지는 순간 사용자가 떠나는 임팩트 도메인과 법률 도메인은, 제품 표면의 안정성과 데이터 정합성이 그대로 사용자 신뢰의 근거가 된다는 점에서 매우 닮아 있다고 봅니다.

또한 로폼이 추구하는 *"기존에 어렵기만한 계약서 등 법률문서를 누구든 쉽고, 편리하게 완성할 수 있도록"*이라는 방향은, 제가 PRD를 직접 쓰면서 사용자의 의사결정 동선·고객사의 신뢰 요구·웹에서의 표현 방식 세 축을 한 문서에서 함께 설계해 온 작업 방식과 직접 연결됩니다. 법률이라는 진입 장벽이 높은 도메인에서 사용자가 *"이 한 장을 완성할 수 있다"*는 자신감을 가지도록 만드는 일은, 단순히 화면을 그리는 일을 넘어 시스템 전체를 함께 보는 사람만이 해낼 수 있는 일이라고 생각합니다.

입사 후 첫 1년은 **로폼의 React/Next.js 코드베이스를 빠르게 읽고, Radix UI + Tailwind CSS 기반 디자인 시스템과 컴포넌트 라이브러리에 일관된 토대를 만드는 일**에 집중하고 싶습니다. myib에서 4개 프로덕트의 인터페이스를 단일 토큰 구조로 수렴시킨 경험은, 빠르게 성장하는 리걸테크 스타트업의 컴포넌트 라이브러리에 그대로 옮겨갈 수 있는 자산이라고 봅니다. 그 토대 위에서 **Chat GLD 같은 AI 기반 문서생성 인터페이스의 사용자 경험을 함께 설계·구현**하는 일에 도전하고 싶습니다. CKEditor 등 리치 텍스트 에디터의 콘텐츠 모델과 LLM API의 출력 흐름을 연결하는 영역은, 평소 AI 협업 워크플로우를 워크플로우에 내재화해 온 작업 방식과 가장 맞닿아 있는 자리라고 생각합니다.

법의 가치가 진짜로 모두에게 실현되는 세상은 결코 혼자서 만들 수 없다고 믿습니다. 팀원들과 긴밀히 협업하고 끊임없이 배우며, 아미쿠스렉스가 지향하는 가치를 사용자가 마주하는 화면에서 안정적으로 구현해 나가는 가장 성실하고 든든한 동료가 되겠습니다.
